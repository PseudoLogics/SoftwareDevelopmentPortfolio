Inventory Management Application

CS 499 Capstone • Software Engineering • Algorithms • Databases

Capstone Artifact

This project represents my final capstone artifact for the Computer Science program. The application began as a basic inventory system and was enhanced to demonstrate growth across software engineering, algorithms, and database design. The final version supports item management, category organization, and real-time search functionality while reflecting improvements in architecture, scalability, performance, and overall user experience.

Software Design & Engineering

I improved the architecture of the application by separating concerns between the user interface, business logic, and database interactions. The RecyclerView adapter was refactored to remove direct database operations, which improved maintainability and reduced unnecessary coupling between components. I also implemented stronger input validation and error handling to increase stability and create a smoother user experience. These changes demonstrate my ability to apply professional software engineering practices to an existing system while making the application more scalable and easier to enhance in the future.

Algorithms & Data Structures

This enhancement focused on improving how data is processed and retrieved within the application. I refined search and filtering functionality to support more efficient item lookup and smoother real-time interaction. By improving the flow of data through the application and organizing it more effectively, the system now performs better as the number of inventory items grows. This demonstrates my understanding of how algorithmic efficiency and data structure choices directly affect usability, responsiveness, and the long-term quality of the solution.

Database Design

The database layer was redesigned using normalization principles to separate items and categories into structured relational tables. Indexes were added to improve query performance, and new attributes such as location and barcode support were introduced to make the system more realistic and practical. These changes improved data integrity, reduced redundancy, and made the application more scalable as a complete inventory solution. This enhancement highlights my ability to design and implement database solutions that are both technically sound and aligned with real-world requirements.
