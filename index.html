Inventory Management Application (Capstone Project)

This application was developed as my CS 499 capstone project and enhanced to demonstrate growth across software engineering, algorithms and data structures, and database design. The system allows users to manage inventory items, categories, and quantities through a modern UI with real-time search and filtering capabilities.

Code Review & Enhancement Plan

This code review analyzes the original version of the application, identifying areas for improvement including inefficient data handling, lack of normalization, and UI limitations. The review outlines planned enhancements and aligns them with software engineering best practices and program outcomes.

Enhancement 1: Software Design & Engineering

I improved the overall architecture of the application by separating concerns between the UI, data handling, and business logic. The RecyclerView adapter was refactored to remove direct database operations, improving maintainability and scalability. Additional error handling and validation were implemented to ensure system stability and a better user experience.

Enhancement 2: Algorithms & Data Structures

The application was enhanced with optimized search and filtering functionality. Efficient querying techniques and indexed lookups were implemented to improve performance when handling larger datasets. Data structures were refined to support faster retrieval and cleaner organization of inventory data.

Enhancement 3: Database Design

The database was redesigned using normalization principles, separating items and categories into distinct tables and introducing relationships between them. Indexes were added to improve query performance, and new fields such as location and barcode support were implemented. This enhancement significantly improved data integrity, scalability, and efficiency.
